# Governance

ORAG is maintained in public. Maintainers make decisions from user impact,
security, operational cost, and compatibility evidence rather than popularity.

## Roles

- Contributors submit issues, discussions, documentation, and pull requests.
- Committers have demonstrated review quality and may merge approved changes.
- Maintainers own releases, security response, compatibility decisions, and
  the contributor/committer roster.

## RFC process

1. Open a GitHub Discussion using the RFC template.
2. Include problem, non-goals, alternatives, compatibility, security,
   operational impact, and validation plan.
3. Leave it open for at least seven days unless a documented security incident
   requires an expedited decision.
4. Record the outcome in `docs/decisions/` before implementation merges.

Security reports follow [SECURITY.md](SECURITY.md), not public RFCs.
